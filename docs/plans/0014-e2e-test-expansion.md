# 計画: E2E テスト拡充

## 動機

現在の E2E テスト（13テスト）は WebRTC ルーム同期に集中しており、
スタンドアロンモードのタイマー操作やメモ機能のテストが不足している。

## 対象

### スタンドアロンモード

- 参加者追加 → タイマー開始 → 次へ → 停止 → リセットの一連フロー
- シャッフル機能
- 不在/出席切り替え
- その場限りの参加者追加・削除

### メモ機能

- 公開メモの入力・保存
- 秘匿メモの入力・保存
- タブ切り替え（公開/秘匿）
- リロード後のメモ復元

### 設定

- グローバル最大時間の変更
- 音量・ミュート設定
- アラート個別 ON/OFF

## 完了条件

- E2E テストが 20 件以上
- スタンドアロンモードの基本フローがカバーされている
- `pnpm run test:e2e` で全テスト通過

## 影響範囲

- `e2e/` ディレクトリに新規テストファイル追加

## 実装結果

- `e2e/timer-standalone.spec.ts` — 6テスト（参加者なし案内、スタート→次へ→ストップ、リセット、シャッフル、その場参加者追加、全員完走）
- `e2e/settings.spec.ts` — 2テスト（設定変更、リロード後保持）
- 既存 13 + 新規 8 = 合計 21 E2E テスト
- 完了日: 2026-03-28

### 注意
- E2E テスト実行は `vp dev` で開発サーバーを起動してから `pnpm run test:e2e`
- メモ機能の E2E は Tiptap エディタ操作が複雑なため未実装（将来対応）
