# dev-loop

## 手順

### 1. コンテキスト読み込み

以下を順番に読む:

1. `.claude/Feedback.md` — 未対応の改善アクションを確認
2. `TODO.md` — 未着手タスクを列挙
3. `.claude/Progress.md` — 進行中タスクがあれば継続、なければ次へ

### 2. タスク選択

TODO.md の「未着手」タスクを対象に、以下の優先順位で1つ選ぶ:

**優先1 — 複利効果**: 他タスクのブロッカー解消・インフラ整備・テスト基盤・ツール改善など、
実施することで後続タスクが楽になるもの。

**優先2 — 若番**: 依存関係がない場合、x.y の数値が小さいもの（例: 21.4 < 22.1）。

選んだタスクを一行で示す: `→ phaseX.Y-xxxx.md を実施します`

### 3. 実装

`docs/plans/phaseX.Y-*.md` を読み、CLAUDE.md のブートシーケンスに従って実装する。
`.claude/Progress.md` の運用ルールに従い、**2段階品質ゲート**（Stage 1: ビルド・Lint・テスト → Stage 2: quality-gate チーム）を完遂する。

### 4. 完了処理

`.claude/Progress.md` の運用ルール「完了処理」ステップ 8〜13 を全て実行し、コミットする。

### 5. ループ継続

`/loop` が次の実行を自動スケジュールするため、1サイクルの処理を終えたら完了とする。
次のサイクルで再びステップ1から再開される。

## 経験の活用
- 実行前に `addf-dev-loop.exp.md` が存在すれば読み、過去の経験（タスク選択判断、よくあるトラブル等）を考慮する
- 実行後、新たな教訓があれば `addf-dev-loop.exp.md` に追記する
